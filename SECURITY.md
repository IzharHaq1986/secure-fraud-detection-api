# Security Policy

## Supported Versions

Only the latest tagged release is supported with security updates.

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

## Reporting a Vulnerability

If a security vulnerability is discovered:

- Do NOT open a public GitHub issue.
- Contact the maintainer directly via:
  - Email: izhar.haq86@yahoo.com

Provide:
- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation (if known)

A response will be provided within 5 business days.

## Disclosure Process

- Vulnerabilities will be acknowledged promptly.
- A fix will be developed and tested.
- A patched release will be tagged.
- The associated SBOM will be updated.
